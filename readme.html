Test WordPress ALC Aperçu Ce projet est un mini-site WordPress développé selon les exigences du test ALC. Il inclut un thème enfant basé sur Twenty Twenty-Five, un bloc Gutenberg personnalisé, un plugin de témoignages et un pipeline CI/CD pour le déploiement. Installation (Locale) Cloner le dépôt : git clone https://github.com/Marilyne14/Aalc-wp-test-Ashley-Fouandjio.git Copier le dossier wp-content dans le dossier wp-content de votre installation WordPress. Activer le thème ALC Child et le plugin ALC Testimonials dans le panneau d'administration WordPress. Configurer une base de données MySQL locale et configurer wp-config.php. Accéder au panneau d'administration à http://localhost/wp-admin avec les identifiants : Nom d'utilisateur : recrutement@alc.test Mot de passe : [Fournir un mot de passe temporaire] Ajouter des témoignages via le panneau d'administration sous "Témoignages". Configurer le slogan du site sous Réglages > ALC. Déploiement Le site est déployé sur Railway avec Docker. Le pipeline CI/CD est configuré dans .github/workflows/deploy.yml et se déclenche à chaque push sur la branche main. URL en ligne : [Fournir l'URL en ligne] Dépôt GitHub : https://github.com/\[votre-username\]/alc-wp-test-\[prenom-nom\] Accès Admin : Nom d'utilisateur : recrutement@alc.test Mot de passe : [Fournir un mot de passe temporaire] Fonctionnalités Thème Enfant (alc-child) : Basé sur Twenty Twenty-Five. Menu principal enregistré. Motif de bloc "Hero ALC" personnalisé. Bloc Gutenberg alc/cta-banner basé sur React, responsive et accessible. Plugin Témoignages : Type de contenu personnalisé : testimonial. Champ méta : rating (1-5). Shortcode : [alc_testimonials] pour afficher 6 témoignages avec un bouton AJAX "Charger plus". API REST : alc/v1/testimonials avec pagination. Page de réglages sous Réglages > ALC pour le slogan du site. Qualité du code : Respect des standards de codage WordPress (vérifié avec phpcs). JS/CSS lintés avec ESLint et Stylelint. Commits clairs et descriptifs. Prérequis WordPress 6.6+ PHP 7.4+ Node.js (pour le développement des blocs) Docker (pour le déploiement) Railway CLI (pour CI/CD) Développement Exécuter npm install dans le dossier blocks/cta-banner du thème pour installer les dépendances. Utiliser wp-scripts build pour compiler le bloc. Utiliser phpcs pour vérifier le code PHP selon les standards WordPress. Notes Le bloc alc/cta-banner est accessible avec des labels ARIA et une navigation au clavier. Le plugin de témoignages utilise l'API REST de WordPress pour le chargement AJAX. L'image Docker est basée sur wordpress:latest et inclut le dossier wp-content.
